Class ObserverEqValue

  • java.lang.Object
    • randoop.contract.ObjectContract
      • randoop.contract.ObserverEqValue

  • public final class ObserverEqValue
extends ObjectContract
    A check recording the value that an observer method returned during execution, e.g. a check recording that a collection's size() method returned 3 when called in particular sequence.

    ObserverEqValue checks are not checks that must hold of all objects of a given class (unlike a check like EqualsReflexive, which must hold for any objects, no matter its execution context). Randoop creates an instance of this contract when, during execution of a sequence, it determines that the above property holds. The property thus represents a regression as it captures the behavior of the code when it is executed.

        A string that can be used as Java source code and will result in the expression being evaluated.

        The string should be formatted as follows: the N-th object that participates in the contract check should be referred to as "xN" (for N one of 0, ... , 9). For example, if the expression of arity 2 represents a call of the equals method between two objects, the comment should be something like "x0.equals(x1)".
