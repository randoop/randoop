randoop.contract

Interface ObjectContract

  • All Known Implementing Classes:
    CheckRepContract, CompareToAntiSymmetric, CompareToEquals, CompareToReflexive, CompareToReturnsNormally, CompareToSubs, CompareToTransitive, EnumValue, EqualsHashcode, EqualsReflexive, EqualsReturnsNormally, EqualsSymmetric, EqualsToNullRetFalse, EqualsTransitive, HashCodeReturnsNormally, IsNotNull, IsNull, ObserverEqValue, PrimValue, ToStringReturnsNormally

    public interface ObjectContract
    An object contract represents a property that must hold of any object of a given class. It is used as part of the oracle (assertion) for a unit test: the oracle expects that every object contract holds. Any sequence of method calls that leads to a failing contract is outputted by Randoop as an error-revealing test case.

    Implementing classes provide two key pieces functionality:

    • A method evaluate(java.lang.Object...)(Object... objects) that determines if the given object(s) satisfy the property.
    • A method toCodeString()() that emits Java code that can be inserted into a unit test to check for the given property.

    See the various implementing classes for examples (for an example, see EqualsReflexive).

      • evaluate

        boolean evaluate(java.lang.Object... objects)
          throws java.lang.Throwable
        Evaluates the contract on the given values.

        When calling this method during execution of a test, Randoop guarantees that objects does not contain any null objects, and that objects.length == getArity().

        This method should return true if the contract was satisfied and false if it was violated.

        Parameters:
        objects - the actual parameters to this contract
        Returns:
        true if this contract evaluates to true for the given values, and false otherwise
        Throws:
        java.lang.Throwable - if an exception is thrown in evaluation

      • evalExceptionMeansFailure

        boolean evalExceptionMeansFailure()
        Communicates to Randoop how to interpret exceptional behavior from the evaluate method.

        If this method returns true, Randoop will interpret an exception that escapes during evaluation as a failure of the contract.

        If the method returns false, Randoop will interpret an exception as passing behavior.

        Returns:
        true if an exception in evaluating this contract should be interpreted as a failure, false otherwise

      • toCodeString

        java.lang.String toCodeString()
        A string that can be used as Java source code and will result in the expression being evaluated.

        The string should be formatted as follows: the N-th object that participates in the contract check should be referred to as "xN" (for N one of 0, ... , 9). For example, if the expression of arity 2 represents a call of the equals method between two objects, the comment should be something like "x0.equals(x1)".

        Returns:
        the code string representation of this contract; must be non-null
