Class ValidityCheckingVisitor

  • java.lang.Object
    • randoop.test.ValidityCheckingVisitor
  • All Implemented Interfaces:
    TestCheckGenerator

    public class ValidityCheckingVisitor
extends java.lang.Object
implements TestCheckGenerator
    A ValidityCheckingVisitor checks for occurrences of exceptions that have been tagged as invalid behaviors as represented by a ExceptionPredicate. Also, either ignores or reports flaky test sequences --- an input sequence that throws an exception in a longer test sequence, despite having run normally by itself. Ignores flaky sequences are classified as invalid. Flaky occurrences of OutOfMemoryError are always treated as invalid.

      • visit

        public TestChecks visit(ExecutableSequence s)
        Generate a TestChecks object for the executed sequence s based on the criteria of this generator.

        Checks validity of a test sequence and creates a InvalidChecks object containing checks for any invalid exceptions encountered. Exceptions are classified by the ExceptionPredicate, but a sequence where an OutOfMemoryError is seen before the last statement is classified as invalid regardless of how OutOfMemoryError is classified by the predicate.
