      • looksLikeObjectToString

        public static boolean looksLikeObjectToString(String s)
        Returns true if the given string looks like it came from a call of Object.toString(); in other words, looks something like "<em>classname</em>@<em>hex</em>". Such strings are rarely useful in generation because they contain non-reproducible hash strings.

        This method is actually more restrictive in what it determines to look like it came from Object.toString(): it deems anything that has a substring matching the pattern @[0-9a-h]{1,8}. Meaning, if it looks like the string contains the "@<em>hex</em>" pattern, the method returns true. This almost always works and is a faster check.

        Parameters:
        s - the string
        Returns:
        true if string appears to be default toString output, false otherwise

      • stringLengthOK

        public static boolean stringLengthOK(String s)
        Returns true if the given string is deemed to be reasonable (i.e. not too long) based on the --string-maxlen=N parameter.

        If Randoop generates tests using strings that are too long, this can result in non-compilable tests due to the JVM's limit on the length of a string.

        A string S is too long if, when printed as code in a generated unit test, it may result in a non-compilable test. In order to determine this, we have to consider not the length of s, but the length of the string that would be printed to obtain s, which may be different due to escaped and unicode characters. This method takes this into account.

        Parameters:
        s - the string
        Returns:
        true if the string length is reasonable for generated tests, false otherwise
        See Also:
        GenInputsAbstract
